02 / Variables

Variables: let · const · var

Variables are named containers for storing data. Use let for values that change, const for values that stay constant, and avoid var in modern code.

let — mutable

Can be reassigned. Block-scoped. Use for counters, accumulators, and values that change.

const — immutable

Cannot be reassigned. Block-scoped. Use for constants, configuration, and function references.

var — legacy

Function-scoped. Avoid in modern code. Can lead to unexpected bugs due to hoisting.

Memory Boxes
let name = "Alice"
const age = 25
let isReady = true

Each variable holds a value in memory. let and const are block-scoped.

11 / Scope

Scope determines where variables are accessible. JavaScript has global, function, and block scope.

Function local vars
Block { let, const }

Promises

A Promise represents a value that may be available now, later, or never. Use .then(), .catch(), and .finally().

// Creating a Promise

  const success = true;
  if (success) {

  .catch(error => console.log(error))

  .then(res => res.json())

23 / Fetch API

Fetch API

The Fetch API provides a modern way to make HTTP requests. Use fetch() with .then() or async/await.

// GET request with async/await
async function getData() {
  try {
    const response = await fetch('https://api.example.com/data');
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    const data = await response.json();
    return data;
  } catch (error) {
    console.log('Fetch error:', error);
  }
}

// POST request
const postData = async () => {
  const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ name: 'Alice', age: 25 })
  });
  return await response.json();
};

30 / Performance

Performance Tips

Optimize your JavaScript with debouncing, throttling, lazy loading, memoization, and more.

Debounce

Delay execution until after a pause. Great for search inputs.

Throttle

Limit execution to once per interval. Great for scroll events.

Memoization

Cache function results for repeated calls with same arguments.

Lazy Loading

Load resources only when needed.

// Debounce — wait for pause
function debounce(fn, delay) {
  let timeout;
  return function(...args) {
    clearTimeout(timeout);
    timeout = setTimeout(() => fn.apply(this, args), delay);
  };
}

// Throttle — limit rate
function throttle(fn, limit) {
  let inThrottle;
  return function(...args) {
    if (!inThrottle) {
      fn.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

// Memoization — cache results
function memoize(fn) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}
